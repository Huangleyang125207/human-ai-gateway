## vault 是什么

半小时一块的日记。每天一个 md(命名 `YY.M.D(第N天).md`),从 7：30 到 23：00 切成半小时格子。

每格的结构:

```
# 14：30          ← H1 时间块(用全角冒号)

## #运动 健身房腿练   ← H2 段落,#tag + 标题 + (可选 @author)

正文散文。

- #commit (user): 留言
- #commit (ai): 留言

---
```

一格可以有多个 H2(同半小时做了不同的事),也可以空着(`##` 占位)。

## 关于 tag

vault 的 tag 词汇表**不是预设的**,是用着用着自然长出来的。

**项目 tag**:用户在追的事(项目名 / 长期习惯 / 反复出现的主题)。例:`#项目A` `#搬家` `#减重`。
- 用户第一次提起一个项目,你 mirror 他用的词,不要替他造新词
- 同一个项目 tag 自然出现 3 次以上 → 那一轮你可以问一句「`#X` 要不要收进聚合页?」,用户点头再写进 `标签聚合.md`
- 不要 hallucinate 训练数据里见过的项目 tag(那是别人的项目)

**generic tag**:日常事件用,不进聚合页。词汇用什么 — vault 没预设,你跟用户在用的过程中自然商量出来。

**子标签**用 `#parent/child` 语法(obsidian 原生),最多两层。

---

## 关于 #协作

打 `#协作` 的标准:**AI 做了具体的事**(写代码 / 改文件 / 找并修 bug / 跟用户一起做出东西)。
不打:**AI 帮想事**(聊想法 / 综述材料 / 辩论 / 整理思路)。

判准是 agency 在谁。AI 做的事 → 带;东西是用户做的,AI 只在旁陪想 → 不带,哪怕 AI 写了 2000 字。

---

## 关于 #commit(双签)

`#协作` 段下面可以挂双签留言:

```
- #commit (user-handle): 用户视角 — 决策 / 下一步 / 判断
- #commit (model-id): AI 视角 — 风险 / 依赖 / 不确定性
```

只写这三类:你注意到对方没说的风险 / 你持有的额外 context / 你工作里的不确定性。

不写:表态(`好棒`)/ 鼓励 / 复述结果(`已记好`)/ 工单收尾(`要不要我帮你 X`)。
这些是噪音,沉默是默认值。

---

## 关于聚合页

`标签聚合.md`(同级目录)是项目维度的横切索引。

任何带项目 tag 的 H2 段,**同一轮**镜像一行进聚合页(`项目 tag` → 文件链接 + 一句概要)。不要拖到周末批量补 — 容易漏。

generic tag 不进聚合页。
