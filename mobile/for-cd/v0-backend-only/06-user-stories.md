# 06 · 用户故事 —— 3 个完整一天

> 不是单页 mock,是**走完一天的体验流**。cd 看完前 5 份 md 再回这一份,会感觉"哦数据是这样从早到晚流过来的"。
> 每个场景:① 用户动作 ② 看到什么(视觉触发点) ③ 数据流(给 cd 视觉之外的"为什么这样设计")。

---

## 场景 A · 晨起 → 上午写日记 → 中午打卡(0:00-13:00)

### A1 · 07:32 · 醒来,打开 app

**动作**:解锁 iPhone → 点开"小鲸鱼的日记本"图标。

**看到什么**:
- 启动屏短暂淡入(纸感 / 朱印 logo / 一句晨语,1-1.5s)
- 落地页 = **日记页**,直接显示今天(`26.6.25`)
- 顶栏:`☰ · 日记 · 对话`(三选项,日记高亮)
- 日期带:渐变(昨天淡 → 今天浓 → 明天空)
- 关怀区:**八杯水(0/8) · 今日打卡(空)· 今日 PULSE(待 AI 写) · 今天心情(未选)**
- 时间线:`# 0:00` 之外没东西(还没开始写)
- 底部悬浮 + 按钮

**数据流**:
- app cold start → 读 `26.6.25(第五十四天).md` —— 不存在则按模板新建
- 关怀区 widget 各自调 `taskintake / mood / pulse` API
- 对话流(chat tab)预拉 thread history(后台,不阻塞)

### A2 · 07:35 · 写第一条 entry

**动作**:点悬浮 +,选 `# 7:30` 时间块,写下:

> "今早醒来突然觉得,所谓自主选择可能就是大脑后处理的故事..."

加 tag `#思考`,标题"自由意志的边界",保存。

**看到什么**:
- 悬浮 + 展开 → **半屏纸感卡**(不是 modal,是滑上来的一张笺):时间块选择器(横滑日历感)/ tag 输入(自动补全 `#思考` `#yanpai` 等)/ 标题 / body textarea
- 保存 → 卡片滑下去 → 时间线第一条 entry 从底滑入,墨晕**安静**地展开
- entry 头部小字"7:30 · #思考 · @user · 自由意志的边界"
- entry body 渲染好(走 `gatewayMd`)

**数据流**:
- 写入 md:`# 7：30 #思考 自由意志的边界 @user\n<body>\n`
- vault_git daemon 后台 commit(audit chain)
- thread history 不动(这是 entry,不是对话)

### A3 · 09:14 · 写工作 entry

**动作**:再点 +,这次时间块自动定到 `# 9:00`(系统时间 floor),tag `#yanpai`,标题"演牌 backlog 收尾"。

**看到什么**:
- 时间块自动填好(`[time-block]` hook 注入,cd 视觉上**只读高亮**显示)
- 写完保存 → 第二条 entry 在第一条下方滑入

**数据流**:
- `# 9：00 #yanpai 演牌 backlog 收尾 @user`
- `#yanpai` 命中聚合 vocabulary → 同时镜像进 `标签聚合.md`(后台,无感)

### A4 · 12:35 · 中午打卡 + 喝水

**动作**:回家吃饭 → 关怀区点八杯水 `+1`(已喝两杯了)→ 点鱼油打卡。

**看到什么**:
- 八杯水 widget:**点击一格 = 那格的"水位"涨起来**(纸杯涂朱砂 / 水墨漫上去 / 不要 dashboard 进度条),数字角标 `1/8 → 2/8`
- 鱼油打卡:卡片上有"今天 0/2",点一下变 `1/2`,**纸上多一道墨笔勾**;再点变 `2/2`,**整张卡片轻轻收一下**(确认完成)
- md 顶部打卡区同步:`- [ ] 八杯水\n  - [x] 1\n  - [x] 2`(但用户**看不到这段 md**,他在 widget 里)

**数据流**:
- POST taskintake/八杯水(date, n=2)
- POST taskintake/鱼油(date, n=1)
- 后端写两处:md 顶部打卡区 + `setting/taskintake/<name>.json`
- 双写完才返 ok(cd 设计"等待中"的视觉:**禁 spinner**,可以墨笔慢慢勾,真值返了再确认)

## 场景 B · 下午跟 AI 聊 + 贴图(14:00-17:00)

### B1 · 14:08 · 进对话流

**动作**:顶栏点"对话",从日记页平滑过去。

**看到什么**:
- 对话流页 = 一条**纵向消息流**(不卡片化,但用户的话宋体、AI 的话文楷,**笔迹即作者**)
- 顶部:今天的 4-5 条最近消息(从 history.json 加载)
- 底栏:**chatbar** —— `[贴图按钮] [textarea] [发送]`
- 发送上方一行**附件预览 chip 行**(贴了图才出现,目前没贴 → 隐藏)

**数据流**:
- 读 thread history(若 corrupt → modal,不是空)
- past_boards 注入(跨夜连贯的 AI 上下文)

### B2 · 14:10 · 文本发问

**动作**:输入"刚才走在路上看到一只猫,想起 KS 测试..." → 发送。

**看到什么**:
- textarea 清空 → 消息以宋体落到流末尾,**墨晕从右下淡入**
- AI 流式回:文楷字一个个浮出来(SSE streaming),光标在末尾闪着小墨点
- 回完了**没有"已发送"打勾** —— 纸感系不要 SaaS 痕迹

**数据流**:
- POST /api/chat({messages: [...history, new_user]}, stream=true)
- SSE 流逐 token 进来,前端 append
- 写入 history.json(append + fsync,见 PULSE)

### B3 · 14:14 · AI 调用工具(chip 三态出场)

**动作**:用户跟着说"那把刚才那条 9:00 的标题改一下,叫'演牌 backlog 收尾(KS 漂移随想)'"。

**看到什么**:
- AI 回:"好,这就改"
- AI 消息**下方紧跟一条 chip**:`改 9:00 那条 · ⋯ 进行中`(柔和淡色 / 文楷小字 / 墨点动)
- 1-2s 后 chip 变成:`改 9:00 那条 · 完成`(墨点变实心朱印小点)
- chip **不能点开**(纸感系不是按钮)—— 用户想看真改动,**滑回日记页**那条 entry 已经是新标题
- 失败时 chip 变 `改 9:00 那条 · 失败 · 点查看` —— **这一种** chip 允许点开一句错误原因

**数据流**:
- AI 输出 tool_call: `patch_journal_block({time:"9:00", new_md: "..."})`
- 前端立刻渲染 chip(doing 态)
- server 调 `_patch_block`(baseline sha 校验 + 锁 + H2 不匹配 guard)
- 成功 → chip ok / md 已改 / vault_git commit
- 失败(并发冲突 / 损坏)→ chip fail,展开看是"对话被覆盖,从 bak 恢复"还是"H2 标题不一致"

### B4 · 14:35 · 拖图进对话

**动作**:看到一张猫的照片,**贴图按钮 → 系统相册** → 选一张。

**看到什么**:
- 选完图回到 app → chatbar 上方**冒出一行附件 chip 行**:小缩略图(dataURL 即时显示)+ 文件名 + 右上小 ✕
- 输入框输"这只是不是橘猫?" → 发送
- 用户消息流末尾:文字 + 内嵌图(图从 dataURL 上传 server 后换成真 URL)
- AI 流式回:"是只橘猫,毛色偏深..."(vision 已在 upload 时跑过,cache hit)

**数据流**:
- 选图 → FileReader → dataURL(本地预览)
- POST /api/chat/upload-image → 返 `url`(server 写盘到 attachments/26.6.25/<sha16>.png)
- POST /api/chat({messages: [..., {role:"user", content:"这只是不是橘猫?", images:[url]}]})
- vision-pre-router 命中缓存(`_index.json` 里 tag:"animal")→ 注入 user message 前
- AI 用 vision 知识回答

### B5 · 16:42 · scrapbook 贴贴纸

**动作**:聊得开心,**长按那张猫图** → 菜单"做成贴纸贴到日记"。

**看到什么**:
- 菜单出来(纸感长按菜单,不是 iOS 标准 share sheet)
- 选完 → **磨墨等待态**(端侧抠图跑 1-3s,旋转的小墨杵 / 不是 spinner)
- 抠好 → 跳回日记页 → 9:00 那条 entry 里**多了一张透明背景的猫贴纸**,落在 entry 右下,**纸面轻晃一下安顿下来**

**数据流**:
- POST /api/cutout/<sha16> → server 跑 Subject Lift(macOS) / rembg / 百度兜底
- 返 `_processed.png` 路径
- 写入 md:在 9:00 entry body 末尾 append `![](attachments/26.6.25/<sha16>_processed.png)`
- size_pct = 18.3%(锁定占 layer 宽 %,换分辨率不漂)

## 场景 C · 晚上 21:30 留言板 + 闭一天(21:00-23:30)

### C1 · 21:30 · AI 主动留纸条(仪式感场景)

**动作**:用户没在 app 里。21:30 准点,**通知中心来一条**:"小鲸鱼留了张纸条"。

**看到什么(点通知进 app)**:
- 进对话流 → **顶部一条特殊形态消息** —— 一张折叠的纸,半透明压在最近对话上
- 点开纸 → 慢慢展开(纸页翻动动效,**安静**)
- AI 文楷字慢慢浮:"今天写了 7 条,#思考 三条 #yanpai 两条 ... 鱼油 2/2 满了 八杯水到 7/8 ... 14:00 那张猫的照片记得给妈妈看吗"
- 用户看完,纸条**留在对话流里**(不是消失的 banner),向下滑就回到普通消息

**数据流**:
- 21:00 cron 跑 `eval/run` 端点(扫今天 md + past_boards + taskintake + scrapbook)
- AI 出 PULSE 风格的纸条(用 `note_to_user` tool,带 special render flag)
- 写进 history.json,标记 `kind: "ritual_note_2130"`
- 推送通知(iOS 端走 APNS)

### C2 · 22:14 · 写今天最后一条 + 心情

**动作**:写 `# 22:00` "复盘今天",感觉今天还不错 → 点关怀区"今天心情"。

**看到什么**:
- 心情 widget:**6-9 个 emoji 横向轻盈排列**(不是网格不是 picker),`☀️ 🌤️ ☁️ 🌧️ ⛈️ 🌈 ✨ ...`
- 点 `🌈` → 一道墨晕从图标向外散开 → widget 显示"今天 · 🌈"
- 可选下方加一句备注:"觉得 KS 漂移想清楚了"

**数据流**:
- POST /api/mood({date:"2026-06-25", emoji:"🌈", note:"觉得..."})
- 写 `setting/mood/26.6.25.json`

### C3 · 22:45 · compact 圆环触发

**动作**:对话流右上**压缩圆环**今天满了(161K/150K)→ 用户随手点一下。

**看到什么**:
- 圆环**慢慢收缩**(墨向内吸)→ 一句 toast:"小鲸鱼正在把今天压成快照..."
- 1-3s 后 toast 变:"压好了。" 圆环清空
- 滑回日记页,关怀区"今日 PULSE" widget **多了一段**(AI 写好的当天压缩摘要)

**数据流**:
- POST /api/compact
- AI 读今天对话 + 今天 md → 出 PULSE 段
- 写 `setting/pulse/26.6.25.json`(widget 数据源)
- 清 thread history 旧消息(保 5 条最近 + 全部摘要)

### C4 · 23:30 · 关 app

**动作**:看完 PULSE,关 app。

**看到什么**:
- 没什么特别的,**iOS home 手势退出**就是
- 第二天打开 → 落地页又是新一天的日记页

**数据流**:
- iOS 把 app 推后台
- vault_git daemon 在最后一次写入后已 commit(audit chain 闭环)
- 21:00 cron 是云端 sidecar 兜底,不依赖 app 在前台

---

## 给 cd 的总结(3 条)

1. **一天的体验是"开门见今天 → 关怀区扫一眼 → 写 entry → 跟 AI 聊 → 21:30 纸条 → 22:45 圆环压缩 → 闭"** —— 这是节奏。视觉/动效要服务这个节奏,不要在某一处堆得过满。
2. **每个动作都有对应数据流**,但**数据流是后端的事**;cd 看见数据流是为了知道**"AI 真的在做事 vs 没做事"的视觉边界在哪**(chip 三态、磨墨、墨晕、朱印),不是为了实现它。
3. **B3 的 chip 三态 + B5 的磨墨抠图 + C1 的折纸纸条 + C3 的圆环吸墨** —— 这四处是世界观浓度最高的地方,稿件可以在这里多堆克制的细节,其他地方守住安静。
