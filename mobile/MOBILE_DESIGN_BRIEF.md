## 核心原则（先读这条，它管住整份 brief）

**移动版 = 桌面版换了交互和 UI 的「同一个程序」，不是新产品。**

能为手机重做的只有两样：**① 交互逻辑（触屏手势）　② UI 呈现（布局 / 视觉 / 动效）**。

**除此之外一切向桌面看齐 —— 照搬，不改、不重新发明、不按「采用率」裁剪：**
功能集 · 数据模型（MD 是真相）· 写作协议（半小时块 / 标签 / 夹批 / authorship 边界）·
AI 行为与人格 · 业务规则（打卡窗口 / 建新天最多 +1 / 双钥匙体系）· 命名与文案口径。

> - cd 只对「长什么样、手指怎么动」负责；「是什么、怎么运作」= 桌面，照抄。
> - 第四节那张清单**不是可选项**，是"桌面有的迟早都要做，差别只是实现顺序"。
> - 任何"移动处理"写法只描述**触屏怎么呈现**，**不得改动**桌面的语义 / 数据 / 规则。
> - 拿不准某功能怎么运作 → 读桌面源码为准，别自己发明。

## 一、这是什么

桌面版 Gateway 已经在跑（Tauri 壳 + 半小时块日记 + 常驻 AI 对话 + 打卡 + 拖图 +
设置市集，内测 ~100 人渠道）。现在要做**手机版**，**iPhone + Android 一套壳**（Capacitor）。

**关键决定：移动版不是把桌面 UI 塞进 webview，是重做一套移动原生交互。**
我们试过克隆桌面——日期随滚动消失、加号点了没反应、卡片排不下，全是触屏死症。
所以推倒，按**传统手机 App 习惯**重做。你这次设计的就是这套移动原生界面。

**魂不变**（细节看桌面 brief，这里只钉死铁律）：安静 · 慢 · 像纸 · 私人 · 深夜低光下舒服。
MD 是真相、AI 是隐形协作者、不要 SaaS/dashboard 感。纸系配色 + 朱砂偶现 + 人宋体/AI 文楷。

## 二、已经对齐死的移动 IA（结构定了，视觉你来）

```
┌──────────────────────────────┐
│ ☰   日记 · 对话               │ ← 顶栏第一行:汉堡最左 + 日记/对话平级切换
│      ‹ 今天·6/13 · 明天 · 灰 › │ ← 第二行:渐变滑动日期带(下详)
├──────────────────────────────┤  ↑ 整条顶栏:下滑隐藏、上滑唤回(让位内容)
│                              │
│   主内容(日记时间线 / 对话流)   │ ← 唯一滚动区
│                              │
├──────────────────────────────┤
│              ( + )           │ ← 底部:日记态=加号;对话态=输入框(下详)
└──────────────────────────────┘
```

钉死的几条：
- **顶栏**：`☰`（最左，所有"非日记非对话"的入口）+ `日记`/`对话` 平级切换。**下滑收起、上滑唤回。**
- **日期带**（渐变滑动，跟 PC 逻辑一致）：横向滑动 5 天窗口、边缘渐隐；今天高亮（朱砂）；
  停在最新日仍露 **2 天未来灰格**；**未创建=灰**；**滑/点未来格=创建那天**，**最多只能建到明天（+1）**。
- **底部栏（你这次改的）**：
  - **日记态 = 一个「+」**。点 + → 弹出**卡片编辑器**（字段：**#tag、时间块、正文**，可含标题）。
    比纯文本输入更贴日记结构。卡片形态/弹出动画/字段排布 = 你设计。
  - **对话态 = 输入框 + 发送**（聊天就该是熟悉的底部输入，回车发、流式出字）。
- **☰ 菜单**（底部抽屉）：设置 · 聚合页 · 小组件 · 历史 · 关于（颗粒度见第四节）。

## 三、触屏交互原则（替掉桌面的鼠标习语）

| 桌面习语 | 移动改成 |
|---|---|
| hover 出工具 / 右键菜单 | **长按** 出操作菜单；或 **左滑** 出删除 |
| 小垃圾桶 icon 删除 | 条目 **左滑删除**（纸的做法，不要 icon 堆） |
| 点元素 @引用进对话 | **长按元素 → 拉进对话**（保留"指给 AI 看"语义） |
| 行内点开直接编辑 | **点条目 → 底部 sheet / 全屏卡片编辑** |
| 拖图到画布定位 | **相册选图 → 抠图 → 贴到某时间块**（touch drag 移动/缩放） |
| 顶部常驻大标题 | **顶栏下滑收起**，内容优先 |

动效方向（延桌面 brief）：呼吸、墨迹晕开、纸页翻动、磨墨等待——**安静**，不要弹跳闪烁。

## 五、硬约束（违反 = 稿子落不了地，工程红线）

1. **vanilla,无 build/npm/框架**（web 层）。数据全走本地 `/api/*`(CC 已写的 shim 拦截层
   读写手机本地 MD + 直连 DeepSeek)。你设计别假设别的数据管线。
2. **markdown 渲染单一入口 `window.gatewayMd()`**（marked+DOMPurify),正文别另造渲染。
3. **纸系 design-tokens 已存在**(色板/字阶/朱砂/人宋·AI文楷)——复用,别引新色值。日/夜双套。
4. **字体走系统 CJK 衬线**(iOS Songti/Kaiti、Android 思源),**零 CDN**(离线/代理下会白屏)。
5. **触屏优先**:点击热区 ≥44pt;无 hover 依赖;手势(滑/长按)要有视觉提示。
6. **safe-area**:刘海/底部 home 条要让位(env(safe-area-inset-*));顶栏/底栏别被吃。
7. **动画安静**:呼吸/墨迹/纸页,不要 dashboard 闪跳;尊重 reduced-motion。
